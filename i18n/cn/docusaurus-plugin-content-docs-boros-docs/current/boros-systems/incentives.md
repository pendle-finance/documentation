# 激励

Boros 运行两个独立的激励计划：一个面向在设定的费率边界内提供流动性的挂单限价单，另一个则将一部分吃单费返还给被吃单方所成交的挂单方。

## 限价单激励

Boros 每周向在订单簿中挂单并随后被吃单方成交的用户分发 PENDLE 激励。其目标是鼓励各市场的有机流动性提供。

### 资格条件

限价单在满足以下条件时有资格获得激励：

- 它作为**挂单订单**下单，即在订单簿中挂单，而非在下单时立即成交。
- 订单费率落在该市场的**激励边界**内（见下文）。

下单时立即成交的限价单将被视为吃单订单，不具备资格。

### 激励边界

每个市场都定义了一个激励边界，即围绕当前中间费率的一个费率区间，挂单限价单必须落在该区间内才有资格。位于该边界之外的订单不获得激励。

该边界因市场而异且是动态的。当订单簿的多头和空头两侧出现流动性失衡时，边界会被非对称地设定：较薄的一侧获得更宽的边界以及按比例更高的奖励，从而形成填补流动性缺口的方向性激励。各市场的具体边界可通过 Boros API 查询。

## 挂单激励

当挂单限价单被吃单方成交时，挂单方将获得该笔交易所支付**吃单费的 20%**作为返佣，每周分发。

### 工作原理

吃单方在每次成交时支付交换费。该费用计算如下：

`吃单费 = 费率 × 名义值（以 YU 计）× 距到期时间（年）`

在该费用中，20% 会传递给订单被成交的挂单方，其余 80% 由协议保留。

### 说明

- 该激励适用于任何已成交的挂单限价单，无论该订单是否同时符合上述限价单激励计划的资格。
- 该激励以用于结算交易的相同抵押品资产计入。
- 挂单方在自己下单时已享受 0% 费率；20% 激励是在此基础上额外叠加的。


## 分发

激励周期每周运行，从周五 00:00 至周五 00:00 UTC。在每个周期结束时，每位符合资格的用户的份额将根据其在该窗口内记录的规模进行计算。

分发最迟于周一 23:59 UTC 之前完成。在处理任何分发之前，团队会审查该周期的活动，以核实所有符合资格的交易量均为合法。对于被标记为操纵或其他恶意的活动，Boros 保留扣留或调整分发的权利。
